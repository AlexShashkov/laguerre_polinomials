\subsubsection*{template$<$class Value\+Type$>$\newline
class ttmath\+::\+Parser$<$ Value\+Type $>$}
Mathematical parser. 

let x will be an input string meaning an expression for converting\+:

x = \mbox{[}+\texorpdfstring{$\vert$}{|}-\/\mbox{]}Value\mbox{[}operator\mbox{[}+\texorpdfstring{$\vert$}{|}-\/\mbox{]}Value\mbox{]}\mbox{[}operator\mbox{[}+\texorpdfstring{$\vert$}{|}-\/\mbox{]}Value\mbox{]}... where\+: an operator can be\+: \texorpdfstring{$^\wedge$}{\string^} (pow) (the heighest priority)

(mul) (or multiplication without an operator -- short mul) / (div) (\texorpdfstring{$\ast$}{*} and / have the same priority)


\begin{DoxyItemize}
\item (add)
\item (sub) (+ and -\/ have the same priority)
\end{DoxyItemize}

\texorpdfstring{$<$}{<} (lower than) \texorpdfstring{$>$}{>} (greater than) \texorpdfstring{$<$}{<}= (lower or equal than) \texorpdfstring{$>$}{>}= (greater or equal than) == (equal) != (not equal) (all above logical operators have the same priority)

\&\& (logical and)

\texorpdfstring{$\vert$}{|}\texorpdfstring{$\vert$}{|} (logical or) (the lowest priority)

short mul\+: if the second Value (Var below) is either a variable or function there might not be an operator between them, e.\+g. "{}\mbox{[}+\texorpdfstring{$\vert$}{|}-\/\mbox{]}\+Value Var"{} is treated as "{}\mbox{[}+\texorpdfstring{$\vert$}{|}-\/\mbox{]}\+Value \texorpdfstring{$\ast$}{*} Var"{} and the multiplication has the same priority as a normal multiplication\+: 4x = 4 \texorpdfstring{$\ast$}{*} x 2\texorpdfstring{$^\wedge$}{\string^}3m = (2\texorpdfstring{$^\wedge$}{\string^}3)\texorpdfstring{$\ast$}{*} m 6h\texorpdfstring{$^\wedge$}{\string^}3 = 6 \texorpdfstring{$\ast$}{*} (h\texorpdfstring{$^\wedge$}{\string^}3) 2sin(pi) = 2 \texorpdfstring{$\ast$}{*} sin(pi) etc.

Value can be\+: constant e.\+g. 100, can be preceded by operators for changing the base (radix)\+: \mbox{[}\#\texorpdfstring{$\vert$}{|}\&\mbox{]} \hypertarget{classttmath_1_1Parser_autotoc_md1}{}\doxysubsection{\texorpdfstring{-\/ hex}{-\/ hex}}\label{classttmath_1_1Parser_autotoc_md1}
\& -\/ bin sample\+: \#10 = 16 \&10 = 2 variable e.\+g. pi another expression between brackets e.\+g (x) function e.\+g. sin(x)

for example a correct input string can be\+: "{}1"{} "{}2.\+1234"{} "{}2,1234"{} (they are the same, by default we can either use a comma or a dot) "{}1 + 2"{} "{}(1 + 2) \texorpdfstring{$\ast$}{*} 3"{} "{}pi"{} "{}sin(pi)"{} "{}(1+2)\texorpdfstring{$\ast$}{*}(2+3)"{} "{}log(2;1234)"{} there\textquotesingle{}s a semicolon here (not a comma), we use it in functions for separating parameters "{}1 $<$ 2"{} (the result will be\+: 1) "{}4 $<$ 3"{} (the result will be\+: 0) "{}2+x"{} (of course if the variable \textquotesingle{}x\textquotesingle{} is defined) "{}4x+10"{} "{}\#20+10"{} = 32 + 10 = 42 "{}10 \texorpdfstring{$^\wedge$}{\string^} -\/\&101"{} = 10 \texorpdfstring{$^\wedge$}{\string^} -\/5 = 0.\+00001 "{}8 \texorpdfstring{$\ast$}{*} -\/\&10"{} = 8 \texorpdfstring{$\ast$}{*} -\/2 = -\/16 etc.

we can also use a semicolon for separating any \textquotesingle{}x\textquotesingle{} input strings for example\+: "{}1+2;4+5"{} the result will be on the stack as follows\+: stack\mbox{[}0\mbox{]}.value=3 stack\mbox{[}1\mbox{]}.value=9
